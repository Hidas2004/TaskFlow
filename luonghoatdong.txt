1.Routes (Lễ tân/Menu):
Bạn nhìn vào Menu và chọn món "Bò bít tết".
Trong code: Routes định nghĩa đường dẫn URL (ví dụ: POST /order). 
Nó chỉ đường cho request biết cần đi đâu.

/////////////////////////////////////////////////////////

2.Handler (Người tiếp tân): Tiếp nhận request, trích xuất ID (số 1). 
Nó không biết database là gì, nó chỉ gọi xuống Service. Sau khi Service trả về kết quả, 
Handler sẽ đóng gói thành JSON để trả về cho Client.

/////////////////////////////////////////////////////////

3.Service (Giám đốc/Logic nghiệp vụ): 
Đây là nơi xử lý "đầu não".
 Ví dụ: Kiểm tra xem user có quyền xem task này không? 
 Task có đang bị khóa không? Nếu ổn, nó gọi xuống Repository.

 /////////////////////////////////////////////////////////////

4.Repository (Thủ kho/Phụ bếp đi chợ):
Đầu bếp hô: "Lấy cho tôi miếng thịt bò".
Thủ kho chạy vào tủ lạnh (Database) lấy thịt mang ra.
Thủ kho không quan tâm thịt đó nấu món gì, họ chỉ biết: Lấy, Cất, Sửa, Xóa.
Trong code: Repo tương tác trực tiếp với Database (dùng SQL hoặc ORM như GORM).
 Nó thực hiện CRUD (Create, Read, Update, Delete).

Database: Nơi lưu trữ dữ liệu thực tế (PostgreSQL, MySQL...).


////////////////////////////////////////////////////////////////////////
Vị trí đặt Interface:

Service Interface: Handler sẽ giữ Interface này.

Repository Interface: Service sẽ giữ Interface này.
